第一篇-Gitee+PicGo图床配置指南：Hexo 博客写作的高效插图解决方案

分类：工具推荐
关键词：图床、Hexo、PicGo、Gitee、Markdown 图片管理

✨ 写在前面

在使用 Hexo 写博客的过程中，插图始终是个绕不开的问题。使用本地相对路径管理图片，虽然 Typora 等 Markdown 编辑器可以预览，但一旦部署上线，就容易出现路径错乱、图片加载失败等问题。
为了解决这一痛点，我选择了 Gitee + PicGo 的图床方案，实现了“一键上传 + 自动复制链接 + Markdown 即用”的高效工作流。

本篇博客将系统介绍如何从零配置这一图床方案，避免踩坑，助你专注写作。

🧰 准备工作

1. 安装工具

  • PicGo 图床上传工具（GitHub地址）

快捷下载链接：PicGo-2.4.0-beta.9

🧱 步骤一：Gitee 仓库准备

  1. 注册并登录 Gitee（gitee.com）
  2. 创建一个新的公开仓库，命名如：picture

注意：这里一定要勾选“开源”，否则图片无法通过链接访问。

  1. 仓库中手动创建一个文件夹用于存图，如：img/，整体仓库架构如下图所示：

📌 Gitee 图床必须使用公开仓库，否则 PicGo 无法上传。

🔐 步骤二：获取 Gitee 的 Token

  1. 打开 Gitee 个人访问令牌 页面

  1. 点击右上角的+生成新令牌，勾选权限 projects和 user_info

  1. 创建后复制 Token，只会显示一次，请妥善保存

⚙️ 步骤三：配置 PicGo 图床参数

下载 PicGo 使用插件

📌 按照图示搜索并下载插件gitee-uploader和quick-capture即可：

  • gitee-uploader插件作用是为了使 PicGo 链接 gitee 库；
  • quick-capture 插件作用是方便通过设置快捷截图按钮实现图片截图快速上传。

打开 PicGo 的图床设置，找到 gitee ，并依次进行以下设置：

  1. 点击+新建一个gitee配置，并将其设为默认图床
  2. 点击图示符号进入编辑页面，并依次进行如下设置：

  • 图床名称：任意，如 “博客图床”
  • repo：你的用户名/picture（如 min-pengyuan/picture）
  • branch：master
  • token：上一步生成的 Token
  • path：img/（你想把图上传到仓库哪个目录下）
  • custompath：选择年月
  • customUrl：可以不填写，默认为https://gitee.com/你的用户名/picture/raw/master

📐 步骤四：设置复制格式为HTML/Markdown等

  1. PicGo 左侧点击【相册】
  2. 展开右上角的小箭头可以看到如下图所示的配置选项，根据需求选择即可：

🔧 步骤五：设置截图快速上传

  1. 找到 PicGo 右侧的 【PicGo设置】，找到设置快捷键，点击点击设置：

  1. 找到快捷截图并上传，这里点击编辑，可以自定义一下快捷键，方便使用，并且不要与电脑中其他软件快捷键冲突，这里建议设置为Ctrl+Alt+X：

  1. 打开 PicGo 主界面，将一张图片拖入上传区域

🚧 常见错误排查

错误提示 原因与解决
Request failed with status code 404 仓库名或路径填写错误，确保 repo 为 用户名/仓库名，且 path 存在
上传成功但无法显示图片 检查 【图床设置】中的 gitee 设置的 path 是否对应于你的仓库中创建的图片文件夹名称
复制内容仍是 ![]() 格式 检查是否修改了【相册】里面的相关设置

🧩 写在最后：为什么选择 Gitee + PicGo？

  • ✅ 图片脱离本地路径管理，更灵活
  • ✅ Markdown 写作所见即所得，编辑器可预览
  • ✅ 图床链接可跨平台使用（知乎、CSDN等）
  • ✅ 完全免费，配置一次即可复用

如果你也在 Hexo 写作过程中被图片管理困扰，试试这个方案，相信你会喜欢上它的流畅体验。如果你还有疑问或想了解 VSCode 一键上传+插入的配置方法，欢迎留言交流！
